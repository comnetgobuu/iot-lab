# ภารกิจเฝ้าระวังภัยพิบัติ  
## ESP32 IoT Workshop สำหรับนักเรียนชั้นประถมศึกษาปีที่ 6

เวิร์กช็อปเรียนรู้การใช้งาน **ESP32** ผ่านภารกิจจำลองระบบเฝ้าระวังภัยพิบัติ  
นักเรียนจะได้ทดลองต่อวงจร อ่านค่าจากเซนเซอร์ ควบคุม LED ใช้ปุ่มกด อ่านค่าโพเทนชิโอมิเตอร์ และแสดงผลผ่านเว็บเบราว์เซอร์

> ระยะเวลาเรียนประมาณ 6 ชั่วโมง  
> ใช้วงจรชุดเดียวตลอดกิจกรรม และเปลี่ยนเฉพาะโปรแกรมในแต่ละภารกิจ

## 🧰 อุปกรณ์ต่อ 1 ทีม

- ESP32 DevKit รุ่น NodeMCU-32S แบบ 38 ขา
- Breadboard
- LED ขนาด 5 มม. 1 ดวง
- ตัวต้านทานสำหรับ LED ขนาด 220–330 โอห์ม
- ปุ่มกด 1 ตัว
- ตัวต้านทานสำหรับปุ่มกดขนาด 5–20 kΩ
- โพเทนชิโอมิเตอร์ 1 ตัว
- เซนเซอร์ DHT11 จำนวน 1 ตัว
- สาย Jumper wire
- คอมพิวเตอร์ Windows 11 พร้อม Arduino IDE

|---|---:|
| ปุ่มกด | GPIO 4 |
| LED | GPIO 5 |
| โพเทนชิโอมิเตอร์ | GPIO 36 |
| DHT11 | GPIO 16 |

## 🧑‍💻 โปรแกรมที่ต้องติดตั้ง

1. ติดตั้ง Driver **CP210x**
2. ติดตั้ง **Arduino IDE 2.3.10**
3. เปิด `File > Preferences`
4. เพิ่ม URL ต่อไปนี้ในช่อง Additional Boards Manager URLs

```text
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

5. เปิด `Boards Manager`
6. ค้นหาและติดตั้ง `esp32 by Espressif Systems`
7. เปิด `Library Manager`
8. ติดตั้ง Library ต่อไปนี้
   - DHT sensor library by Adafruit
   - Adafruit Unified Sensor by Adafruit
9. เลือกบอร์ด

```text
Tools > Board > ESP32 > NodeMCU-32S
```

## 🚀 ภารกิจในเวิร์กช็อป

### ภารกิจที่ 1: ไฟและปุ่มกด
เรียนรู้ Digital Input และ Digital Output โดยใช้ปุ่มกดควบคุม LED

### ภารกิจที่ 2: หมุนแล้วเห็นผล
อ่านค่า Analog จากโพเทนชิโอมิเตอร์ผ่าน GPIO 36

### ภารกิจที่ 3: เซนเซอร์วัดอากาศ
อ่านค่าอุณหภูมิและความชื้นจาก DHT11

### มินิเกม: กดให้ไว ก่อนภัยมา
วัดเวลาปฏิกิริยาของผู้เล่นจากการกดปุ่มเมื่อ LED ติด

### ภารกิจพิเศษ: เว็บเซิร์ฟเวอร์
ให้ ESP32 สร้าง Wi-Fi และแสดงค่าจากเซนเซอร์ผ่านเว็บเบราว์เซอร์ โดยไม่ต้องใช้อินเทอร์เน็ต

### เกมจบ: หมุนให้ตรงค่า
หมุนโพเทนชิโอมิเตอร์ให้ได้ค่าใกล้เคียงค่าเป้าหมายมากที่สุด

## ⚠️ ข้อควรระวัง

- ตรวจสอบตำแหน่งขา `3.3V`, `GND` และ GPIO ก่อนจ่ายไฟ
- ห้ามต่อ LED โดยไม่มีตัวต้านทาน
- DHT11 ต้องต่อไฟและขาสัญญาณให้ถูกต้อง
- ก่อน Upload โปรแกรม ให้เลือก Board และ Port ให้ตรงกับอุปกรณ์
- ในภารกิจเว็บเซิร์ฟเวอร์ แต่ละทีมต้องตั้งชื่อ Wi-Fi ไม่ซ้ำกัน
- หาก Upload ไม่ผ่าน ให้ตรวจสอบ Driver, สาย USB และหมายเลข COM Port
